data-if 非表示分岐のフォーム送信除外 デモ

同じ name="value" の入力を設定型（モード）ごとに data-if で 出し分けるデモです。data-if が false の分岐は DOM から消えず data-if-false 属性付きで残りますが、フォーム値収集（data-click-form 等）の 対象からは自動的に除外されます。表示中の分岐の値だけが収集され、配列累積や値の競合は起きません。

割引モード:

例: 100（円）

例: 0.5（50% 割引）

収集されるフォーム値:
mode = {{mode}} value = {{value}}

ポイント

  • モードを切り替えると、表示される入力（固定額／割合）が data-if で入れ替わります。
  • value は常に単一値です。非表示分岐の同名入力は収集されないため、["100","0.5"] のような配列累積は発生しません。
  • 送信値からは除外されますが、同名の入力要素自体は DOM 上に残ります。

マークアップ例

<form data-bind='{"mode":"fixed","value":"100"}'>
  <input type="radio" name="mode" value="fixed" checked> 固定額
  <input type="radio" name="mode" value="ratio"> 割合

  <!-- 表示中の分岐だけが収集される -->
  <div data-if="mode === 'fixed'">
    <input type="text" name="value">
  </div>
  <!-- 非表示（data-if-false）のあいだは収集対象外 -->
  <div data-if="mode === 'ratio'">
    <input type="text" name="value">
  </div>
</form>

セレクタで「1要素だけ」を前提にしたい場合は、入力を1つに統一し type / step / max 等を {{}} 式で切り替える運用も検討してください。
